## 三、需要改进的问题

### 🔴 高优先级（建议近期修复）

#### 问题1：主题代码提交到仓库

**当前问题**：
- `themes/butterfly/` 整个目录都在 git 仓库里
- 未来升级困难，代码耦合
- 难以区分"主题本身修改"和"定制配置"

**解决方案**（推荐方案2）：

**方案一：使用 git submodule（推荐给懂 git 的人）**
```bash
# 移除当前主题
git rm -r themes/butterfly

# 添加为 submodule
git submodule add https://github.com/jerryc127/hexo-theme-butterfly themes/butterfly
git commit -m "Use butterfly as git submodule"
```

**方案二：不提交主题代码（最简单）**
```bash
# 1. 添加到 .gitignore
echo "themes/butterfly/" >> .gitignore

# 2. 记录主题版本
echo "hexo-theme-butterfly: v5.5.5-b1" >> .gitmodules-record.txt

# 3. 移除并重新安装（可选）
git rm -r --cached themes/butterfly
git commit -m "Do not track theme code"
```

**推荐方案二**，原因：
- 你主要通过 `_config.butterfly.yml` 配置，没有修改主题代码
- 升级简单：重新下载即可
- 仓库更干净

### 🟡 中优先级（1年内处理）

#### 问题2：文章都在单一目录

**当前**：所有文章在 `source/_posts/`

**风险**：
- 第5年，500篇文章时，找文件开始困难
- 没有归档，只有标签分类

**解决方案**：
近期不需要改，当文章数超过 200 时，可以按年份归档：
```
source/_posts/
├── 2024/
├── 2025/
├── 2026/
└── ...
```

**注意**：修改 `_config.yml` 的 `new_post_name` 配置即可。
