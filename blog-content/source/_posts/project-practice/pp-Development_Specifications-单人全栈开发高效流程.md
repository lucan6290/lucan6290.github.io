# 单人全栈开发高效流程（最快交付版）

> 适用于：独立全栈开发工程师 / 单人负责一个系统板块
>
> 核心目标：**最快交付 + 最少返工**

---

## 一、核心原则

单人开发遵循：

```
业务流 → 数据流 → API → 后端 → 前端 → 优化
```

避免：

* ❌ 先前端
* ❌ 先数据库建表
* ❌ 上来直接写代码

---

## 二、黄金开发流程（推荐顺序）

### Step 1：业务流程设计（最重要）

**投入时间：约 30%**

需要明确：

* 用户是谁？
* 用户第一步做什么？
* 页面如何跳转？
* 数据从哪里来？
* 最终产出结果是什么？

**输出物：业务流程图（手绘即可）**

示例：

```
登录
 ↓
首页
 ↓
专家查询
 ↓
详情
 ↓
申请 / 收藏
```

> 没有流程图，后期极易返工。

---

### Step 2：数据库设计（从业务倒推）

原则：**从页面需求倒推数据模型**

方法：

```
页面需要什么数据？
↓
这些数据属于谁？
↓
形成数据表
```

输出物：

* 简单 ER 关系
* 核心字段即可

示例：

```
user
expert
expert_tag
application
```

初期不要过度设计：

* ❌ 索引优化
* ❌ 冗余字段
* ❌ 分库分表
* ❌ 性能调优

---

### Step 3：API 接口清单（效率核心 ⭐）

先定义接口，而不是先写代码。

示例：

```
POST   /login
GET    /experts
GET    /expert/{id}
POST   /apply
```

为每个接口定义：

* 请求参数
* 返回结构

示例返回：

```json
{
  "id": "",
  "name": "",
  "title": ""
}
```

完成后即拥有：

* 产品设计
* 架构设计
* 数据设计
* 前后端契约

---

### Step 4：后端开发（优先于前端）

原因：

* API 是系统核心
* 前端只是展示层
* 前端可 Mock
* 修改前端成本低

推荐顺序：

```
Entity
 ↓
Mapper
 ↓
Service
 ↓
Controller
```

完成标准：

* Swagger 能调通全部接口

此时系统约完成 **70%**。

---

### Step 5：接口自测

使用工具：

* Apifox
* Postman
* curl

验证完整业务流程：

```
登录
查询
新增
修改
删除
```

目标：

> 不依赖前端即可跑通业务。

---

### Step 6：前端开发

此时前端主要为：**数据绑定与交互实现**。

开发流程：

```
页面骨架
↓
API 接入
↓
状态管理
↓
交互优化
```

Vue 推荐结构：

```
Page → API → Store → Component
```

---

### Step 7：前后端联调

由于 API 已稳定：

* 联调成本低
* 问题集中

通常 1～2 天可完成。

---

### Step 8：权限与日志（最后添加）

正确顺序：

```
系统跑通
↓
权限控制
↓
操作日志
```

避免权限系统导致频繁改动。

---

### Step 9：部署与环境

开发过程中同步准备：

* Dockerfile
* Nginx
* 环境变量配置

理想状态：

```
git pull
docker compose up
```

即可启动系统。

---

## 三、终极流程总览

```
① 业务流程图
② 页面列表
③ 数据库设计
④ API 清单 ⭐⭐⭐⭐⭐
⑤ 后端开发
⑥ API 自测
⑦ 前端开发
⑧ 联调
⑨ 权限日志
⑩ 部署上线
```

---

## 四、效率工具建议

| 类型   | 推荐工具         |
| ---- | ------------ |
| 接口设计 | Apifox       |
| 数据建模 | draw.io      |
| ORM  | MyBatis Plus |
| Mock | Swagger      |
| 部署   | Docker       |
| 状态管理 | Pinia        |
| 权限模型 | RBAC 模板      |

---

## 五、高手效率技巧（自动化）

目标：减少重复 CRUD 编码。

```
接口定义 → 自动生成前端 → 自动生成 CRUD
```

实践方式：

* OpenAPI 自动生成前端 SDK
* MyBatis Plus 代码生成器
* Admin 模板复用

可减少约 **60% 手写代码量**。

---

## 六、推荐开发节奏（参考）

| 阶段        | 建议时间占比 |
| --------- | ------ |
| 业务设计      | 30%    |
| 数据/API 设计 | 20%    |
| 后端开发      | 25%    |
| 前端开发      | 15%    |
| 联调与部署     | 10%    |

---

> 结论：**先想清业务与 API，再写代码，是单人开发最快路径。**
