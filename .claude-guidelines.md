# 箓川码笺 —— 项目开发规范

> 本文档供 Claude Code 等 AI 助手阅读，了解本项目的规范和约定。

## 项目概览

- **项目名称**：箓川码笺
- **项目类型**：Hexo 静态博客
- **技术栈**：Hexo 8.x + Butterfly 5.5.5-b1
- **部署**：GitHub Pages + GitHub Actions
- **作者**：lucan
- **语言**：中文为主，技术术语可用英文

## 核心规范

### 1. 文章文件命名规范

遵循 `[分类前缀]-[具体主题].md` 格式：

| 分类 | 前缀 | 示例 |
|------|------|------|
| 前端技术 | `fe-` | `fe-vue3-组件通信.md` |
| 后端技术 | `be-` | `be-springboot-快速上手.md` |
| AI/Agent | `ai-` | `ai-agent-入门笔记.md` |
| 算法 | `algo-` | `algo-二叉树遍历.md` |
| 工具使用 | `tool-` | `tool-hexo-博客搭建.md` |
| 成长随笔 | `growth-` | `growth-求职经历总结.md` |

详细规范见：`source/_posts/growth-博客文章命名规范.md`

### 2. 目录结构

```
blog/
├── source/
│   ├── _posts/          # 博客文章（重要）
│   ├── _drafts/         # 草稿
│   ├── about/           # 关于页面
│   ├── categories/      # 分类页
│   └── tags/            # 标签页
├── themes/
│   └── butterfly/       # Butterfly 主题（不要直接修改！）
├── _config.yml          # Hexo 主配置
├── _config.butterfly.yml # Butterfly 主题配置（重要）
└── package.json
```

### 3. 文章 Front Matter

每篇文章必须包含以下内容：

```yaml
---
title: 文章标题
date: 2026-05-22 15:30:00
categories:
  - [一级分类, 二级分类]
tags: [标签1, 标签2, 标签3]
description: 文章简介（100-200字）
cover: /img/covers/封面.jpg  # 可选
sticky: 100  # 可选，置顶优先级
---
```

### 4. 写作规范

- **语言**：中文为主，清晰易懂
- **格式**：Markdown，合理使用标题、列表、表格、代码块
- **代码块**：注明语言，示例要正确可运行
- **引用链接**：重要参考资料给出链接
- **图片**：放在与文章同名的文件夹中，使用相对路径

### 5. 协作规则

#### 必须遵守

1. ✅ 先阅读 `_config.butterfly.yml`，了解主题配置结构
2. ✅ 不要直接修改 `themes/butterfly/` 下的文件（会被覆盖）
3. ✅ 重大变更（如删除文章、修改配置）先确认再执行
4. ✅ 新增文章后，运行 `npm run build` 验证可以正常构建
5. ✅ 保持文章文件名和内容的一致性

#### 推荐做法

- 先了解现有文章的风格再创建新文章
- 复杂问题拆分成小步骤，逐步执行
- 执行破坏性操作前，先看 git diff

## 常用命令

| 命令 | 说明 |
|------|------|
| `npm run server` | 启动本地预览服务器（http://localhost:4000） |
| `npm run build` | 生成静态文件到 `public/` |
| `npm run clean` | 清理缓存和 `public/` |
| `npm run deploy` | 部署到 GitHub Pages |
| `npx hexo new "文章标题"` | 创建新文章 |
| `npx hexo new page "页面名称"` | 创建新页面 |

## Git 与部署

- 推送到 `main` 分支会自动触发 GitHub Actions 部署
- Commit 信息使用 Conventional Commits 格式：`feat: xxx` / `fix: xxx` / `docs: xxx`
