## 前言

Vibe Coding（氛围编码）是一种全新的编程范式，通过与 AI 助手协作，让开发者更专注于创意和架构设计，而不是繁琐的代码细节。

Claude Code 是 Anthropic 官方推出的 AI 编程工具，集成在桌面端、VS Code 等环境中，支持多语言、多任务、智能上下文理解，是 Vibe Coding 的绝佳搭档。

---

## 一、什么是 Claude Code

### 1.1 核心特点

| 特性 | 说明 |
|------|------|
| 🌐 **多平台支持** | 桌面端（Windows/macOS/Linux）、VS Code 插件 |
| 📚 **强大上下文** | 理解整个项目结构，跨文件推理 |
| 🔧 **工具调用** | 自主执行命令、编辑文件、运行测试 |
| 🎯 **安全设计** | 沙箱环境，权限可控，操作可见 |
| 🤝 **人机协作** | 开发者主导，AI 辅助，分工明确 |

### 1.2 适用场景

- ✅ 新项目快速搭建
- ✅ 功能模块开发
- ✅ 代码重构与优化
- ✅ Bug 排查与修复
- ✅ 文档编写
- ✅ 技术学习与探索

## 二、安装 Claude Code

### 2.1 方式一：桌面端（推荐新手）

#### Windows / macOS / Linux

1. 访问 [Claude Code 官网](https://claude.ai/code)
2. 下载对应系统的安装包
3. 双击安装，一路 Next
4. 启动后登录 Anthropic 账号

### 2.2 方式二：VS Code 插件

1. 打开 VS Code
2. 点击左侧「扩展」图标
3. 搜索「Claude Code」
4. 点击「安装」
5. 安装完成后，点击左侧 Claude Code 图标登录

### 2.3 方式三：CLI（命令行）

```bash
# 使用 npm 安装
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

## 三、基础配置

### 3.1 账号设置

1. 登录 Anthropic 账号
2. （可选）配置 API Key（高级功能）
3. 设置默认工作目录

### 3.2 项目初始化

#### 打开现有项目

```bash
# 在项目根目录启动
cd /path/to/your/project
claude .
```

或在桌面端 / VS Code 中直接打开项目文件夹。

#### 创建新项目

直接告诉 Claude 你的需求，它会帮你搭建：

> "帮我创建一个基于 Vue 3 + Vite 的前端项目"

## 四、核心功能快速上手

### 4.1 对话式开发

Claude Code 的核心是自然语言对话。用清晰的指令描述你的需求。

**示例对话：**

```
用户：帮我在 src/components/ 下创建一个 UserCard 组件，显示用户头像、名字、邮箱
Claude：好的，我来创建这个组件...
[创建文件 src/components/UserCard.vue]
Claude：已创建 UserCard 组件，需要我帮你在 App.vue 中引入使用吗？
```

### 4.2 项目结构理解

Claude 会自动扫描项目，理解：
- 技术栈（Vue/React/Node 等）
- 目录结构
- 代码风格
- 依赖关系

你可以问：

> "这个项目的技术栈是什么？"
> "帮我分析一下项目结构"
> "代码风格有什么特点？"

### 4.3 多文件编辑

Claude 可以同时处理多个文件，适合：
- 创建关联的多个组件
- 重构跨文件的代码
- 添加新功能（涉及前端 + 后端）

**示例：**

> "创建一个登录功能，包含：
> 1. LoginForm.vue 组件
> 2. 对应的 API 调用
> 3. 路由配置"

## 五、基础使用技巧

### 5.1 提问的艺术

#### ✅ 好的指令特点

- **具体明确**：说明文件名、路径、技术选型
- **拆分任务**：复杂功能拆成多个小步骤
- **提供参考**：给出类似代码或风格示例

**示例对比：**

```
❌ 不太好：帮我写个登录页面
✅ 比较好：在 src/views/Login.vue 下创建登录页面，使用 Element Plus UI，包含邮箱/密码输入、记住我、登录按钮
```

### 5.2 常用指令参考

| 场景 | 示例指令 |
|------|---------|
| 创建文件 | "在 src/utils/ 下创建 request.js，封装 axios" |
| 修改文件 | "修改 UserCard.vue，添加一个编辑按钮" |
| 重构代码 | "重构这个函数，让它更简洁易读" |
| 排查 Bug | "帮我看看为什么这个接口返回 500" |
| 添加注释 | "给这段代码添加清晰的注释" |
| 编写测试 | "为这个组件写单元测试" |
| 代码审查 | "帮我 Code Review 一下这个文件" |

### 5.3 项目规范传达

在项目开始时，先告诉 Claude 你的技术规范：

> "在这个项目中，我们遵循以下规范：
> 1. 使用 TypeScript
> 2. 组件用 Composition API
> 3. 文件名用 kebab-case
> 4. 先确认再执行重大变更"

Claude 会在后续开发中遵守这些约定。

## 六、安全与注意事项

### 6.1 Claude Code 的安全设计

- 🔒 **权限控制**：需要你明确授权才能执行命令
- 👀 **操作透明**：所有文件变更可见，可以预览
- ↩️ **撤销方便**：不满意可以一键回退
- 🚫 **不发送敏感数据**：（可选）配置忽略文件

### 6.2 .gitignore 与 .claudeignore

建议在项目根目录创建 `.claudeignore`（类似 `.gitignore`），排除敏感文件：

```
# 敏感配置
.env
.env.local
*.key

# 依赖目录
node_modules/
dist/

# IDE 配置
.idea/
.vscode/
```
