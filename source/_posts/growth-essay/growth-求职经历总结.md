## 写在前面

最近开始投技术实习岗位，踩了不少坑，也积累了一些经验。这篇文章把我的求职过程整理出来，希望能帮到同样在找实习的小伙伴。

## 一、简历打磨

### 核心原则：STAR 法则

简历上的项目经验，用 **STAR 法则** 来写：

- **S**ituation — 背景是什么
- **T**ask — 你要做什么
- **A**ction — 你做了什么
- **R**esult — 取得了什么成果

```markdown
❌ 差：参与了 xxx 项目的后端开发
✅ 好：负责 xxx 项目的用户模块，使用 Spring Boot + Redis 实现了
   高性能接口，QPS 从 200 提升到 2000，响应时间降低 60%
```

### 排版建议

1. **一页纸** — 实习生简历不要超过一页
2. **突出重点** — 技术栈、项目成果加粗
3. **PDF 格式** — 不要发 Word，格式会乱
4. **检查错别字** — 至少检查三遍

{% note warning %}
手机号、邮箱一定要再三检查！我有一个朋友因为简历上邮箱写错了一位数，错过了好几家公司的面试通知。
{% endnote %}

## 二、投递渠道

| 渠道 | 优点 | 缺点 |
|------|------|------|
| **Boss 直聘** | 反馈快、直接沟通 | 公司质量参差 |
| **牛客网** | 内推多、笔试题库 | 竞争激烈 |
| **实习僧** | 专注实习 | 大公司少 |
| **GitHub** | 展示技术实力 | 被动等待 |
| **公司官网** | 直达 HR | 回复慢 |

**我的策略**：Boss 直聘主动投 + 牛客网找内推 + 公司官网同步投，三条线并行。

## 三、面试准备

### 技术栈复习

```
必考内容：
├── 数据结构与算法（LeetCode Hot 100 必刷）
├── 计算机网络（TCP/UDP、HTTP、DNS）
├── 操作系统（进程线程、内存管理）
└── 项目经验（能讲清楚技术选型和难点）
```

### 常见面试问题

**Q：自我介绍怎么讲？**

不要背简历，用 **3 句话结构**：

> 我是谁（学校/专业）+ 我会什么（技术栈）+ 我做过什么（1-2 个亮点项目）

**Q：项目中遇到的难点？**

选一个真正有挑战的问题，讲清楚：
1. 问题是什么
2. 为什么难
3. 你怎么解决的
4. 学到了什么

### 面试心态

1. **不会很正常** — 实习生容错率高，关键是态度
2. **反问环节要珍惜** — 问团队技术栈、业务方向，表示你对这个岗位真的感兴趣
3. **及时复盘** — 每次面试后记录问题和回答，不断优化

## 四、踩过的坑

### 坑 1：简历投了 50 份没回应

**问题**：用的通用简历，一份简历投所有岗位

**解决**：根据 JD（岗位描述）微调简历，突出匹配的技能

### 坑 2：面试紧张，脑子空白

**问题**：面试官问了一个基础问题，突然忘了

**解决**：平时多在牛客网练习「手撕代码」，培养肌肉记忆

### 坑 3：忘了问面试结果

**问题**：面完一周没消息，不知道挂了还是没处理

**解决**：面试后 3-5 个工作日，礼貌发 HR 微信或邮件跟进

## 总结

找实习这件事，心态比能力更重要。被拒不代表你不行，只是匹配度不够。保持学习，持续投递，总有一个 offer 在前面等你。
