# 博客背景样式不一致问题修复计划

## 问题分析

### 现象描述
- **首页**：正常显示深色背景、代码元素动画、站点标题文字（使用 `default-cover.svg`）
- **其他页面**（归档、分类、标签、关于）：背景为纯色，未加载首页的背景样式

### 根本原因

通过分析 Butterfly 主题的 `themes/butterfly/layout/includes/header/index.pug` 模板逻辑：

```pug
case globalPageType
  when 'home'
    - top_img = returnTopImg(theme.index_img)
  when 'archive'
    - top_img = returnTopImg(theme.archive_img)
  when 'tag'
    - top_img = theme.tag_per_img && theme.tag_per_img[page.tag] || returnTopImg(theme.tag_img)
  when 'category'
    - top_img = theme.category_per_img && theme.category_per_img[page.category] || returnTopImg(theme.category_img)
  when 'page'
    - top_img = page.top_img || theme.default_top_img
  default
    - top_img = page.top_img || theme.default_top_img
```

**关键发现**：
1. 首页（`home` 类型）：使用 `index_img`，已正确配置为 `/img/default-cover.svg`
2. 归档页面（`page` 类型，不是 `archive` 类型）：使用 `page.top_img || theme.default_top_img`
3. 分类页面（`page` 类型）：使用 `page.top_img || theme.default_top_img`
4. 标签页面（`page` 类型）：使用 `page.top_img || theme.default_top_img`
5. 关于页面（`page` 类型）：使用 `page.top_img || theme.default_top_img`

**问题所在**：
- 归档、分类、标签、关于这些独立页面在 Butterfly 主题中被识别为 `page` 类型
- 这些页面的 front matter 中没有设置 `top_img`
- `default_top_img` 在 `_config.butterfly.yml` 中为空（未配置）
- 因此这些页面没有背景图片，显示为纯色背景

### 配置现状

当前 `_config.butterfly.yml` 中的相关配置：
```yaml
index_img: /img/default-cover.svg      # 首页背景 ✅ 已配置
archive_img: /img/default-cover.svg    # 归档列表页背景 ✅ 已配置
tag_img: /img/default-cover.svg        # 标签列表页背景 ✅ 已配置
category_img: /img/default-cover.svg   # 分类列表页背景 ✅ 已配置
default_top_img:                        # 默认背景 ❌ 未配置（为空）
```

## 解决方案

### 方案一：配置 `default_top_img`（推荐）

**优点**：
- 一次配置，全局生效
- 所有未单独设置背景的页面都会使用默认背景
- 维护简单，符合主题设计理念

**实施步骤**：
1. 在 `_config.butterfly.yml` 中添加 `default_top_img: /img/default-cover.svg`
2. 重新生成站点验证效果

### 方案二：为每个页面单独设置 `top_img`

**优点**：
- 灵活性高，每个页面可以有不同的背景
- 更精细的控制

**缺点**：
- 需要修改多个文件
- 维护成本高
- 新建页面时容易忘记设置

**实施步骤**：
1. 在 `source/about/index.md` 的 front matter 中添加 `top_img: /img/default-cover.svg`
2. 在 `source/categories/index.md` 的 front matter 中添加 `top_img: /img/default-cover.svg`
3. 在 `source/tags/index.md` 的 front matter 中添加 `top_img: /img/default-cover.svg`
4. 对于归档页面，需要创建 `source/archives/index.md` 并设置 `top_img`

## 推荐实施方案一

### 具体修改

修改 `_config.butterfly.yml` 文件，在图片设置部分添加 `default_top_img` 配置：

```yaml
# --------------------------------------
# 图片设置
# --------------------------------------
favicon: /img/favicon.svg

avatar:
  img: /img/avatar.svg
  effect: true

# 首页顶部大图
index_img: /img/default-cover.svg
archive_img: /img/default-cover.svg
tag_img: /img/default-cover.svg
category_img: /img/default-cover.svg

# 默认背景图片（用于所有未单独设置背景的页面）
default_top_img: /img/default-cover.svg
```

### 验证步骤

1. 运行 `npm run clean` 清理缓存
2. 运行 `npm run server` 启动本地服务器
3. 访问以下页面验证背景是否一致：
   - 首页：http://localhost:4000/
   - 归档：http://localhost:4000/archives/
   - 分类：http://localhost:4000/categories/
   - 标签：http://localhost:4000/tags/
   - 关于：http://localhost:4000/about/

### 预期效果

所有页面都将显示与首页一致的背景效果：
- 深色渐变背景（深蓝到墨绿）
- 代码元素动画（代码雨效果）
- 巨大的"箓"字水印
- 流动的河流线条动画

## 补充说明

### 关于 `archive_img` 的说明

虽然配置了 `archive_img`，但这个配置只在 `globalPageType === 'archive'` 时生效。而实际的归档页面（`/archives/`）是一个独立页面，其类型是 `page`，所以不会使用 `archive_img`。

### 关于页面类型的说明

Butterfly 主题的页面类型判断：
- `home`：首页
- `post`：文章详情页
- `page`：独立页面（包括归档、分类、标签、关于等）
- `tag`：标签列表页（`/tags/xxx/`）
- `category`：分类列表页（`/categories/xxx/`）

### 扩展：为不同页面设置不同背景

如果将来想为不同页面设置不同的背景，可以：
1. 创建不同的背景图片
2. 在各页面的 front matter 中设置 `top_img`，例如：
   ```yaml
   ---
   title: 关于
   top_img: /img/covers/about-bg.svg
   ---
   ```

## 总结

问题的根本原因是 `default_top_img` 未配置，导致独立页面（归档、分类、标签、关于）没有背景图片。

最简单的解决方案是在 `_config.butterfly.yml` 中添加 `default_top_img: /img/default-cover.svg`，这样所有页面都会使用统一的背景样式。
